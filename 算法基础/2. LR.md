## 2. 工业界的应用方式

LR把所有特征搞成one-hot,然后拼成一个超长稀疏向量
举例
用户特征：

性别：男 / 女

年龄段：少年 / 中年 / 老年

id：1 / 2 / 3 / …

如果这个用户是男 + 中年 + id=2

那么向量就是1   0   0    1    0    0     1    0 ...


<img width="1001" height="497" alt="image" src="https://github.com/user-attachments/assets/c6116b48-0508-4e47-bd6d-2c680760ec48" />

理论上要把所有的w都存下来，然后用w和用户的x向量相乘，但是由于特征的稀疏性，有很多的0，工业上为了节省资源，只存非0的。

如何把大部分参数都压缩成0？ 只保留重要的，用FTRL（工业界）

FTRL（follow the regularized leader)

特点： 

支持L1 稀疏

支持在线学习（来一个样本更新一次）

适合超高维稀疏数据

## 3. 推荐的本质是复读机

被点击的item的权重会变大，没被点击的item权重不大

只会记住“谁点过谁”，不会推理，不会泛化，不会自动学交叉关系

如果是人工设计交叉特征的话，1. 累  2. 想不全面

解决这个问题，增加泛化特征（能够帮助模型理解结构的特征）复读机问题的根本原因是，模型维度太少，无法解释行为，

记忆模型升级为归因模型


## 4. 扩展： LR的非线性（MLR)

线性模型的劣势，

多层纯线性= 一层线性， 光堆线性层没有意义

sigmoid可以增加非线性能力，但还是太弱了。

## MLR
采用多个LR，把特征空间分为m个部分，每一个部分用一个LR模型，此时模型可以看成是多个LR组成的模型。
增大m可以提高非线性拟合能力，但可能发生过拟合，增加计算，存储的开销


## LR的损失函数（最重要）

<img width="420" height="157" alt="image" src="https://github.com/user-attachments/assets/c396bc18-c699-4d6b-9010-43f8d95a74f0" />

最大似然就是选一个θ， 让这个数据很合理，而不是这个数据几乎不可能发生，
LR用于分类的时候，损失函数是交叉熵损失函数，是通过极大似然函数推导出来的

用BCE 而不用MSE 原因

MSE（Mean squared error)更新很慢，

<img width="402" height="184" alt="image" src="https://github.com/user-attachments/assets/5ef3ded2-22b9-4fbe-8fbe-228f7cf7ad81" />
<img width="498" height="146" alt="image" src="https://github.com/user-attachments/assets/c137d5cd-fbde-4fce-b84d-fb69e32a14df" />

多乘了一个σ′(z)，当模型的输出接近0或者1的时候，σ′(z) 就会非常小，接近0，使得求得的梯度很小，损失函数收敛很慢

对于BCE(Binary Cross Entropy) 交叉熵损失（对数似然函数）
<img width="765" height="143" alt="image" src="https://github.com/user-attachments/assets/32de654b-d9de-4a79-9e39-189b39f71366" />

机器学习里面是最小化损失，MLE是最大化似然。

<img width="383" height="130" alt="image" src="https://github.com/user-attachments/assets/c0b93696-5886-4370-a8b2-a10add43a3a5" />

可以看到，使用交叉熵损失函数时，参数w更新的梯度值之和预测误差和模型的输出有关，不会受到sigmoid函数的影响。


MSE 服从高斯分布，和高斯分布本质不符合，BCE服从伯努利分布，和二分类的label很契合。

平方差损失函数是非凸函数，交叉熵函数是凸函数。   二阶导>0则为凸函数

BCE在逻辑回归等线性模型中是凸函数，能保证找到全局最优解，而MSE与sigmoid等非线性激活函数结合后，损失函数可能非凸，存在多个局部极小值，优化困难。但在非线性强度深的深度神经网络中，BCE也不是凸函数了。
