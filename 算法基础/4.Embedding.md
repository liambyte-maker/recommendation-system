# Embedding

## 1. 提出的背景
1. LR（Logistics regression)胜在记忆力

   推荐算法面临的经典问题无非是记忆与扩展
   
   LR 非常擅长把训练数据中高频出现的模式直接学成大权重，并在相同模式出现时迅速给出高预测。

2. 记忆的原理：
   每个特征都有一个分数，当一个样本过来，会得到一个总分，然后过一个sigmoid

   权重大不一定重要，印尼我特征尺度可能不一样

   LR强于记忆，能够记住，不会推理

   比如中国人 → 爱吃饺子    美国人 → 爱吃火鸡
它会记住：

<中国人, 饺子> 是高分
<美国人, 火鸡> 是高分

LR不发掘新模式，反之它能够通过FTRL，剔除一些得分较低的罕见模式，既避免过拟合，又降低了参数量

# 2.如何让LR模型更有扩展性

解决思路

把细粒度概念拆成粗粒度特征（人工构造embedding)

加入抽象特征之后，新样本即使具体组合没有出现过，只要它再抽象空间中与历史样本相似，就能获得分数

工作量大，劳神费力

很多时候，特征受到工程师的业务水平，理解能力，创意水平的制约

## 3. embedding 的作用

embedding 把离散id 变成连续向量

Embedding作用

1. 降维

2. 提升扩展能力

模型可能做点积，如果向量方向接近，点积大，分数高

原始LR 精确匹配， Embedding  模糊匹配（相似性匹配）

Embedding实现过程

类别型特征，直接搞成Embeding

数值型特征，1. 搞成类别型特征，再转为embedding,等频率分桶（每个桶里数据一样多）或者等距分桶

Embedding 长度怎么确定？

如果太短，表达能力太弱，如果太长，参数过多，容易过拟合。

embedding表征学习的难点

1. 参数规模大，embedding 会占模型存储体积的百分之80以上

2. 收敛速度慢，embedding层参数太多，导致训练缓慢，，预训练Embedding可以缓解这个问题

4. 长尾
   训练数据大部分由头部热门的商品产生，冷门商品的数据很少。冷启动的item训练数据比较少，导致训练推荐效果差。

## 4. 得到Embedding的三种方式

端到端学习两种方式

不提前算embedding， 不人工设计，用最终的任务来训练

预测点击率 → 算loss → 反向传播 → 更新embedding

热训练

假设去年已经训练过一个推荐模型，新模型可以直接用旧模型的embedding当初始值，然后微调

收敛快，表达更稳定

2. 序列建模

   word2Vec   Item2Vec

   出现在一起的东西，embedding靠近

   图模型

   比如有关系网络

   用GCN,GAT,让有连接的节点embedding变得相似

   
## Embedding 的不同类型

共享Embedding

原因： 数据少，单独训练太少，如果共享，所有数据一起训练，embedding被更新次数更多
如果不共享，参数量会变成N倍

解决冷启动+长尾问题

独占式的Embedding

在数据量充足的情况下，独占式Embedding相比与共享式Embedding，可以避免Embedding之间的相互拉扯

比如 点击和购买

点赞和举报，他们对embedding的要求不同

## 6. 工业界处理Embedding的方式

书本里面学的是 one-hot 向量 × W = embedding

但工业界不会这么做，

因为 id可能有几亿个， one-hot非常巨大，矩阵会无限扩展

工业界做法

embedding_matrix[id]

embedding lookup table
当新商品出现，分配一行新向量，随机初始化1

embedding的矩阵不会无限大，因为会预先设定一个最大空间

hash映射会产生冲突，当embedding表很大，冲突概率小，冷门数据本来少，共享embedding影响不大

很长时间没出现过的id embedding信息就被清楚了

embedding长度设定， 16，32，64都可以，一般就是取2的次方
