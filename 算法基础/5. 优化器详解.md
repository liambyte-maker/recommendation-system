# 优化器详解释


# 1. 背景

优化器的作用：

深度学习中，优化器用于调整模型的参数（权重，偏置，embedding),来使得损失函数最小

鞍点在一个方向上是最低点， 但是在另一个方向上，是最高点

因为深度学习神经网络，参数维度非常高，在高空间，鞍点数量远远多于真正的局部最小值

梯度下降会认为已经收敛，但其实不是最低点

## 区分鞍点和最小值

看二阶导数，

如果所有方向曲率都>0, 最小值

如果有的方向>0, 有的<0, 鞍点


## 2. 常考优化器机器考点

梯度下降类

最常见的一类优化器，通过计算loss相对于模型参数的梯度，沿着梯度下降的方向更新参数，以最小化损失函数

SGD(stochastic Gradient Descent 随机梯度下降）

优点： 效率高，更新速度快

缺点： 容易在训练中产生波动，用于每次参数更新时采用的数据量小，造成梯度更新时震荡服读大，容易收到异常值的影响，在最优解附近会有较大波动，但多数情况下都是向着梯度减小的方向

BGD(Batch Gradient Descent 批量梯度下降）

每次迭代中使用整个数据集计算梯度（将所有样本的梯度求和，然后根据梯度和对参数进行更新）

优点； 收敛稳定，梯度更新比较平滑

缺点：训练开销大

MBGD(Mini-Batch Gradient Descent)

每次只使用一部分进行参数更新，平衡了批量梯度下降和随机梯度下降的优缺点）

## 梯度下降的优缺点

优点：

1. 简单直观，梯度下降原理简单，计算损失函数的梯度，沿着梯度最速下降方向进行优化

2. 课扩展性强，梯度下降可以在不同规模和复杂度下使用，通过调整批量大小，可以适应不同数据集的规模

3. 良好的收敛性，对于凸优化问题，梯度下降可以保证收敛到全局最优解，对于非凸问题，梯度下降也可以找到局部最优解

缺点： 

1. 收敛速度慢，梯度下降，尤其是批量梯度下降，收敛速度慢

2. 容易陷入局部最优，在非凸问题中，梯度下降容易陷入局部最优解

3. 学习率敏感，梯度下降依赖与超参数学习率的选择，如果学习率太小，收敛速度慢，如果学习率太大，梯度下降可能无法收敛，甚至发散

4. 如果我们的数据是稀疏的，我们希望对出现频率低的特征进行步子大一点的更新

   
batchsize:

小batch，更新频繁，随机性，有助于跳出局部最优

缺点： 抖动大

大batch: 梯度更稳定，收敛更平滑

缺点计算慢，泛化可能变差
